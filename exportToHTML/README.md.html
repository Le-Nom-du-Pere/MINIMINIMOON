11   ## Responsibility Detection Component
12   
13   A Python module for detecting responsibility entities in Spanish text using spaCy's Named Entity Recognition (NER) combined with lexical pattern matching.
14   
15   ### Files
16   - **responsibility_detector.py** - Core responsibility detection with spaCy NER and pattern matching
17   - **test_responsibility_detector.py** - Comprehensive test suite for responsibility detection
18   
19   ### Features
20   - **SpaCy NER Integration**: Uses spaCy's pre-trained Spanish models for PERSON and ORGANIZATION entity recognition
21   - **Government Entity Priority**: Higher confidence scores for government institutions like "alcaldía", "secretaría", "programa"
22   - **Official Position Detection**: Recognizes institutional roles and positions with pattern matching
23   - **Lexical Fallbacks**: Comprehensive pattern matching for cases where NER misses institutional language
24   - **Confidence Scoring**: Combines NER confidence with lexical pattern strength for final factibility assessment
25   - **Overlap Handling**: Merges overlapping entities intelligently, preferring higher confidence and government entities
26   
27   ### Usage
28   ```python
29   from responsibility_detector import ResponsibilityDetector
30   
31   # Initialize detector
32   detector = ResponsibilityDetector()
33   
34   # Analyze text
35   text = "La Alcaldía Municipal coordinará con la Secretaría de Salud el programa de vacunación."
36   result = detector.calculate_responsibility_score(text)
37   
38   print(f"Factibility Score: {result['factibility_score']:.3f}")
39   print(f"Government entities detected: {result['has_government_entities']}")
40   
41   for entity in result['entities']:
42       print(f"- {entity.text} ({entity.entity_type.value}, confidence: {entity.confidence:.3f})")
43   ```

45   ## Embedding Model Component
46   
47   A robust embedding model implementation that automatically falls back from MPNet to MiniLM if the primary model fails to load, ensuring uninterrupted operation.
48   
49   ### Files
50   - **embedding_model.py** - Core embedding model with fallback mechanism
51   - **test_embedding_model.py** - Comprehensive test suite for embedding model
52   - **example_usage.py** - Demo and usage examples

55   - **Automatic Fallback**: Tries MPNet (768-dim) first, falls back to MiniLM (384-dim) if loading fails
56   - **Exception Handling**: Comprehensive error handling around model initialization
57   - **Model-Dependent Configuration**: Automatically adjusts batch sizes and parameters based on the loaded model
58   - **Similarity Calculations**: Built-in cosine similarity computation
59   - **Factory Pattern**: Easy model instantiation with `create_embedding_model()`
60

77   ## Factibilidad Scoring Component
78   
79   A Python module for detecting and scoring text segments based on the presence of baseline, target, and timeframe patterns in Spanish text.
80   
81   ### Files
82   - **factibilidad/pattern_detector.py** - Core pattern detection logic
83   - **factibilidad/scoring.py** - Scoring algorithms and analysis
84   - **test_factibilidad.py** - Test cases and examples
85   
86   ### Features
87   - **Pattern Detection**: Identifies three types of patterns in Spanish text:
88     - Baseline indicators (línea base, situación inicial, punto de partida, etc.)
89     - Target indicators (meta, objetivo, alcanzar, lograr, etc.)
90     - Timeframe indicators (dates, quarters, relative time expressions)
91   - **Proximity-Based Clustering**: Groups patterns that appear within a configurable distance window
92   - **Factibilidad Scoring**: Calculates scores based on individual pattern presence, complete pattern clusters, proximity bonuses, and pattern density bonuses
93   
94   ### Usage
95   ```python
96   from factibilidad import PatternDetector, FactibilidadScorer
97   
98   # Basic pattern detection
99   detector = PatternDetector()
100  matches = detector.detect_patterns(text)
101  
102  # Calculate factibilidad scores
103  scorer = FactibilidadScorer(proximity_window=500)
104  result = scorer.score_text(text)
105  
106  print(f"Score: {result['total_score']}")
107  print(f"Clusters found: {result['cluster_scores']['count']}")
108  ```

195  ## Statistical Interpretation (DAG Validation)
196  
197  ⚠️ **Important**: DAG validation tests structural acyclicity, not causal validity.
198  
199  - **P-value**: Probability of observing acyclic structure in random subgraphs under the null hypothesis
200  - **Lower p-values**: Suggest the observed acyclicity is unlikely to occur by chance alone
201  - **Higher p-values**: Indicate the structure could reasonably arise from random processes
202  
203  ### Limitations
204  1. **Not a causal test**: Validates graph structure, not causal relationships
205  2. **Domain expertise required**: Statistical significance ≠ causal validity
206  3. **Interpretation context**: Results must be interpreted within domain knowledge
207  4. **Subgraph sampling**: Tests random subsets, not the full graph structure

240  
241  ### Entity Types
242  - `PERSON`: Individual names detected by NER
243  - `ORGANIZATION`: Generic organizations
244  - `GOVERNMENT`: Government institutions (higher priority)
245  - `POSITION`: Official roles and positions
246  
247  ### Confidence Scoring Hierarchy
248  1. **Government institutions** (highest priority): 0.8-1.0 confidence
249  2. **Official positions**: 0.7-0.9 confidence  
250  3. **Persons via NER**: 0.6-0.8 confidence
251  4. **Generic organizations**: 0.5-0.7 confidence
252  
253  ### Pattern Categories
254  
255  #### Government Institutions (High Priority)
256  - alcaldía, secretaría, programa, ministerio
257  - gobernación, municipalidad, ayuntamiento
258  - instituto nacional/municipal/departamental
259  - dirección, departamento administrativo
260  
261  #### Official Positions
262  - alcalde/alcaldesa, secretario/secretaria
263  - ministro/ministra, director/directora
264  - coordinador/coordinadora, jefe/jefa
265  - responsable, encargado/encargada

267  #### Fallback Institutional Terms

277  ## Architecture
278  
279  The module uses a multi-stage approach:
280  
281  1. **NER Processing**: Extract PERSON and ORG entities using spaCy

283  3. **Entity Merging**: Handle overlaps and duplicates
284  4. **Confidence Calculation**: Apply hierarchical scoring based on entity types
