36   /* 
37   A common CUDA interface for ATen. 
38    
39   This interface is distinct from CUDAHooks, which defines an interface that links 
40   to both CPU-only and CUDA builds. That interface is intended for runtime 
41   dispatch and should be used from files that are included in both CPU-only and 
42   CUDA builds. 
43    
44   CUDAContext, on the other hand, should be preferred by files only included in 
45   CUDA builds. It is intended to expose CUDA functionality in a consistent 
46   manner. 
47    
48   This means there is some overlap between the CUDAContext and CUDAHooks, but 
49   the choice of which to use is simple: use CUDAContext when in a CUDA-only file, 
50   use CUDAHooks otherwise. 
51    
52   Note that CUDAContext simply defines an interface with no associated class. 
53   It is expected that the modules whose functions compose this interface will 
54   manage their own state. There is only a single CUDA context/state. 
55   */
56   
57   /**

60   inline int64_t getNumGPUs() {
61       return c10::cuda::device_count();

64   /** 
65    * CUDA is available if we compiled with CUDA, and there are one or more 
66    * devices.  If we compiled with CUDA but there is a driver problem, etc., 
67    * this function will report CUDA is not available (rather than raise an error.) 
68    */
69   inline bool is_available() {
70       return c10::cuda::device_count() > 0;
71   }
