tunable.py 
1    r""" 
2    This module exposes a TunableOp interface. 
3     
4    Some operations, such as GEMMs, could be implemented using more than one library 
5    or more than one technique. For example, a GEMM could be implemented for CUDA or 
6    ROCm using either the blas or blasLt libraries. Further, ROCm's rocblas and 
7    hipblaslt libraries allow the user to query for all possible algorithms and then 
8    choose one. How does one know which implementation is the fastest and should be 
9    chosen? That's what TunableOp provides. 
10    
11   Enabling TunableOp and Tuning Separately 
12   ======================================== 
13    
14   The TunableOp feature is enabled separately from enabling the tuning phase 
15   itself. Enabling TunableOp means that PyTorch will replace any standard 
16   operators with their Tunable implementations. Any call to a TunableOp first 
17   checks whether it has already been tuned for the given operator inputs. If so, 
18   it will immediately call the tuned operation; no further tuning will take place 
19   even when the tuning setting is enabled. Instead if no tuning result is found, 
20   and tuning is enabled, the TunableOp will benchmark every registered 
21   implementation of that operator for the given set of inputs and select the 
22   fastest. 
23    
24   File Input and Output 
25   ===================== 
26    
27   The first time any TunableOp is invoked, the internal database of tuned 
28   operations will be prepared by attempting to read the results from the given 
29   file. The default filename is 'tunableop_results.csv'. To support tuning when 
30   multiple GPUs are used across multiple processes, the GPU device ordinal is 
31   automatically inserted into the filename to avoid multiple processes overwriting 
32   the same file. 
33    
34   If tuning is enabled and new tunings are discovered during the course of your 
35   workload, it will also write out to this same filename with all tunings, both 
36   the ones it read in at startup as well as the new ones found at runtime. This 
37   can be used, for example, to build up a tunings file across many workloads by 
38   reusing the same file. The output file is automatically created when the 
39   application terminates. This behavior can be controlled by the C++ and Python 
40   APIs but not the environment variables. 
41    
42   Assuming you specified a filename, you'll end up with a CSV file with contents 
43   like so:: 
44    
45     Validator,PT_VERSION,2.2.0 
46     Validator,ROCM_VERSION,6.0.0.0-12969-1544e39 
47     Validator,HIPBLASLT_VERSION,0.6.0-a9c5cc7 
48     Validator,ROCBLAS_VERSION,4.0.0-72e57364-dirty 
49     GemmTunableOp_float_NT,nt_25088_4096_64,Gemm_Hipblaslt_1219,1.262 
50     GemmTunableOp_float_NT,nt_4096_4096_64,Gemm_Rocblas_1216,0.033 
51    
52   Note the "Validator" lines. If you change a library version, or ROCm version, or 
53   PyTorch version, TunableOp will detect this and reject the tunings file because 
54   the prior tunings are likely affected by other software changes. 
55    
56   The remaining lines are the tuned solutions for each TunableOp encountered 
57   during your execution. Each line consists of 4 comma-separated fields: operator 
58   name, operator parameters, solution name, and average execution time. The 
59   execution time is an optional field. The CSV file can be edited, but with 
60   caution. For example, the solution name (field 3) can be changed to "Default" 
61   and it will fall back to the original PyTorch untuned implementation. Or, in the 
62   case of ROCm's hipBLAS or hipBLASLt libraries, if you know the specific solution 
63   index you can override the solution that TunableOp selected by replacing the 
64   value. The operator name and parameters (fields 1 and 2) are internally named 
65   and should not be modified. In the case of GemmTunableOp, field 1 indicates the 
66   datatype and whether the inputs are transposed (T) or not (N) and field 2 
67   indicates the M, N, K input shapes. 
68    
69   There is an option to enable verbose output but it is only recommended for 
70   debugging purposes. This will produce a lot of diagnostic messages but may be 
71   useful to see if TunableOp is being used at all. Otherwise, TunableOp is 
72   completely silent, besides file output, unless there is a warning or error 
73   during its use. The verbose option is only available by setting the environment 
74   variable PYTORCH_TUNABLEOP_VEROBSE=1. 
75    
76   A Note on Tuning Behavior, Warmup, and Cache Effects 
77   ==================================================== 
78    
79   Tuning an operator consists of iterating through the list or registered 
80   implementations and profiling each one. The profile is established by running a 
81   single implementation in a loop multiple times and taking the average execution 
82   time. There is also an optional warmup phase prior to tuning that can help with 
83   reaching stable power states by the hardware. During tuning of a workload the 
84   various hardware caches will more likely produce hits than when not tuning. 
85   There are options for flushing the instruction cache and rotate the input tensors 
86   which might help produce a more faithful profile of the tuned operator as if the 
87   operator were run within a larger workload instead of in a tight, repetitive loop. 
88    
89   By default, each possible solution for a given operator will be run for either 
90   100 iterations or as many iterations that can be run within 30ms, whichever is 
91   smaller, and its average execution will be calculated. The fastest solution 
92   among all that were successfully profiled will be chosen. A profile might fail 
93   if the given solution doesn't achieve the same accuracy as the default 
94   implementation or if the solution returns an error code. 
95    
96   Current Tunable Operators 
97   ========================= 
98    
99   TunableGemm for ROCm 
100  -------------------- 
101   
102  Currently only a TunableGemm for ROCm is implemented. Note that CUDA builds of 
103  PyTorch will function correctly when using TunableOp but the only solution 
104  available to CUDA builds is the 'Default' implementation i.e. the original 
105  cuBLAS default, now called through TunableOp. Any call to at::cuda::blas::gemm() 
106  or ::bgemm() will be routed through TunableOp when enabled. Calling gemm() for a 
107  given set of input arguments (transa, transb, m, n, k) will attempt to use the 
108  fastest available implementation across both rocblas and hipblaslt. 
109   
110  Offline Tuning 
111  ============== 
112   
113  Motivation 
114  ---------- 
115  There are several use cases for offline tuning. 
116   
117  One use case involves a workload with a high-memory utilization, where regular tuning might lead to running out of memory. 
118   
119  Another use case is for compute-intensive workloads. In such cases, it is more resource-efficient to collect 
120  the GEMMs for the workload once and then tune repeatedly with different tuning parameters or libraries. 
121   
122  Workflow 
123  -------- 
124  There are basically two steps: 
125  1) Set the environment variables to collect the untuned GEMM and this will generate ``tunableop_untuned0.csv``: 
126   
127  .. code-block:: python 
128   
129     PYTORCH_TUNABLEOP_ENABLED=1 
130     PYTORCH_TUNABLEOP_TUNING=0 
131     PYTORCH_TUNABLEOP_RECORD_UNTUNED=1 
132     ... 
133   
134  2) Run a Python script that reads the ``tunableop_untuned0.csv`` and generates the ``tunableop_results0.csv``, like this: 
135   
136  .. code-block:: python 
137   
138     import torch.cuda.tunable as tunable 
139     import os 
140   
141     os.putenv('PYTORCH_TUNABLEOP_ENABLED', '1') 
142     os.putenv('PYTORCH_TUNABLEOP_TUNING', '1') 
143     os.putenv('PYTORCH_TUNABLEOP_RECORD_UNTUNED', '0') 
144     tunable.tune_gemm_in_file("tunableop_untuned0.csv") 
145   
146   
147  It is also possible to take multiple untuned files and distribute the GEMMs for tuning to multiple GPUs 
148  within a single node. In the first step, the GEMMs are first gathered and duplicate GEMMs are eliminated. 
149  Next, the GEMMs are distributed to different GPUs for tuning. After all GEMMs are tuned, the results from 
150  all the GPUs are then gathered into a single file whose base filename has ``_full0`` appended to it 
151  (for example ``tunableop_results_full0.csv``). Finally, this new file, containing the gathered results, will be 
152  duplicated N times, once for each GPU as convenience to the user will run the workload with the tuned 
153  configuration on N GPUs. 
154   
155  .. code-block:: python 
156   
157     if __name__ == "__main__": 
158         num_gpus = 8 # number of GPUs that will be used during the tuning process 
159         tunable.mgpu_tune_gemm_in_file("tunableop_untuned?.csv", num_gpus) 
160   
161  Note that the usage of the ``mgpu_tune_gemm_in_file`` API is different from its single GPU counterpart 
162  (``tune_gemm_in_file``). The body of the Python script that calls the API must be wrapped in ``main()`` as shown 
163  due to the use of concurrent futures module. The argument to ``mgpu_tune_gemm_in_file`` must contain a wild card 
164  expression (``?`` or ``*``) to generate the list of untuned files containing the GEMMs to be processed. The ``num_gpus`` 
165  must between 1 and the total number of GPUs available. 
166   
167  Tuning Context 
168  ============== 
169   
170  The behavior of TunableOp is currently manipulated through environment 
171  variables, the C++ interface of at::cuda::tunable::getTuningContext(), or the 
172  torch.cuda.tunable python interfaces. The environment variables take precedence 
173  over any setting you manipulate using the C++ or Python APIs. 
174   
175  Environment Variable Interface 
176  ------------------------------ 
177  Environment variables are cached the first time they are read. You cannot use the 
178  environment variable interface programmatically since the settings become fixed. 
179  Use the C++ or Python APIs instead.

365      r"""Process multiple untuned results file and return a set with duplicates removed."""
366      unique_gemm_entries = set()  # set will avoid duplicates
367  
368      for file_path in glob.glob(filename_pattern):
369          with open(file_path) as file:
370              for line in file:

372                      unique_gemm_entries.add(line)

374      return unique_gemm_entries

377  def _gather_tunableop_results() -> None:
378      r"""Gather results from multiple tunableop results file and create a single file."""
379      gemm_lines = set()
380      validator_lines = []
381  
382      # Need to allow for the possibility that results filename was
383      # set with the Python API instead of with environment variable.
384      # Also possible that results filename was not set at all.
385      # There are several test cases to check, but ultimately we
386      # need a glob-able expression
387      results_filename = get_filename()  # Note empty string could be returned here
388  
389      if (
390          results_filename is not None and results_filename != ""
391      ):  # Case were the Python API was used to set the filename
392          dot_pos = results_filename.find(".")
393          if dot_pos != -1 and dot_pos > 0:
394              # Replace the character just to the left of the dot
395              filename_pattern = (
396                  results_filename[: dot_pos - 1] + "?" + results_filename[dot_pos:]
397              )
398          else:
399              filename_pattern = ""  # Needed to make linter happy
400      else:  # Case where the environment variable was used to set the filename.
401          results_filename_env = os.getenv("PYTORCH_TUNABLEOP_FILENAME")
402          if results_filename_env is None or results_filename_env == "":
403              filename_pattern = "tunableop_results?.csv"
404          elif "%d" in results_filename_env:
405              filename_pattern = results_filename_env.replace("%d", "?")
406          else:
407              filename_pattern = results_filename_env.replace(".", "?.")
408  
409      assert "?" in filename_pattern
410  
411      FirstFile = False
412      matching_files = glob.glob(filename_pattern)
413      num_matching_files = len(matching_files)
414      for file_path in matching_files:
415          with open(file_path) as file:
416              for line in file:
417                  if line.startswith("Validator"):
418                      if not (FirstFile):
419                          # Only read Validator from first file
420                          validator_lines.append(line)
421                  else:
422                      gemm_lines.add(line)
423  
424          FirstFile = True
425  
426      output_file = filename_pattern.replace("?", "_full0")
427  
428      with open(output_file, "w") as out_file:
429          for line in validator_lines:
430              out_file.write(line)
431          for line in gemm_lines:
432              out_file.write(line)
433  
434      # Create num_matching_copies of the results file
435      for i in range(1, num_matching_files):
436          duplicate_file = output_file.replace("0", str(i))
437          shutil.copy(output_file, duplicate_file)

440  def _create_matrices(
441      m: int,
442      n: int,
443      k: int,
444      lda: int,
445      ldb: int,
446      ldc: int,
447      transA: bool,
448      transB: bool,
449      dtypeA: torch.dtype,
450      deviceid: str,
451      dtypeB: Optional[torch.dtype] = None,
452      randn: bool = True,
453      subMatrix: bool = False,
454  ) -> tuple[torch.Tensor, torch.Tensor]:
455      r"""Helper function for _process_single_offline_gemm. 
456      Creates matrices that are then consumed by one of the Torch GEMM APIs. 
457      """
458      # Fill parameters set for use with ScaledGEMM
459      fillA = 0.25
460      fillB = 0.75
461  
462      if dtypeB is None:
463          dtypeB = dtypeA
464  
465      if subMatrix:
466          # User reference for understanding leading dimension:
467          # https://github.com/Reference-LAPACK/lapack/blob/master/BLAS/SRC/dgemm.f
468          # TO DO: According to lines 108 - 133, there is no lower bound on rowsA,
469          # but there is a restriction on rowsB. Using this formula for now as it
470          # seems to work for all UTs.
471          rowsA = rowsB = max(ldc, k)
472  
473          if randn:
474              matA = torch.randn(rowsA, lda, dtype=dtypeA, device=deviceid)
475              matB = torch.randn(rowsB, ldb, dtype=dtypeA, device=deviceid)
476          else:
477              matA = torch.full((rowsA, lda), fillA, dtype=dtypeB, device=deviceid)
478              matB = torch.full((rowsB, ldb), fillB, dtype=dtypeB, device=deviceid)
479  
480          subA = matA[:k, :m].t() if transA else matA[:m, :k]
481          subB = matB[:n, :k].t() if transB else matB[:k, :n]
482          return subA, subB
483      else:
484          if randn:
485              matA = (
486                  torch.rand(k, m, dtype=dtypeA, device=deviceid).t()
487                  if transA
488                  else torch.rand(m, k, dtype=dtypeA, device=deviceid)
489              )
490              matB = (
491                  torch.rand(n, k, dtype=dtypeB, device=deviceid).t()
492                  if transB
493                  else torch.rand(k, n, dtype=dtypeB, device=deviceid)
494              )
495          else:
496              matA = (
497                  torch.full((k, m), fillA, dtype=dtypeA, device=deviceid).t()
498                  if transA
499                  else torch.full((m, k), fillA, dtype=dtypeA, device=deviceid)
500              )
501              matB = (
502                  torch.full((n, k), fillB, dtype=dtypeB, device=deviceid).t()
503                  if transB
504                  else torch.full((k, n), fillB, dtype=dtypeB, device=deviceid)
505              )
506          return matA, matB

509  def _create_batch_matrices(
510      m: int,
511      n: int,
512      k: int,
513      b: int,
514      lda: int,
515      ldb: int,
516      ldc: int,
517      transA: bool,
518      transB: bool,
519      dtype: torch.dtype,
520      deviceid: str,
521      subMatrix: bool = False,
522  ) -> tuple[torch.Tensor, torch.Tensor]:
523      r"""Helper function for _process_single_offline_gemm. 
524      Creates batch matrices that are then consumed by one of the Torch GEMM APIs. 
525      Similar to _create_matrices but for 3D batch matrices. 
526      """
527      if subMatrix:
528          # User reference for understanding leading dimension:
529          # https://github.com/Reference-LAPACK/lapack/blob/master/BLAS/SRC/dgemm.f
530          # TO DO: According to lines 108 - 133, there is no lower bound on rowsA,
531          # but there is a restriction on rowsB. Using this formula for now as it
532          # seems to work for all UTs.
533          rowsA = rowsB = max(ldc, k)
534  
535          matA = torch.randn(b, rowsA, lda, dtype=dtype, device=deviceid)
536          matB = torch.randn(b, rowsB, ldb, dtype=dtype, device=deviceid)
537  
538          subA = matA[:b, :k, :m].transpose(1, 2) if transA else matA[:b, :m, :k]
539          subB = matB[:b, :n, :k].transpose(1, 2) if transB else matB[:b, :k, :n]
540          return subA, subB
541      else:
542          matA = (
543              torch.rand(b, k, m, dtype=dtype, device=deviceid)
544              if transA
545              else torch.rand(b, m, k, dtype=dtype, device=deviceid)
546          )
547          matB = (
548              torch.rand(b, n, k, dtype=dtype, device=deviceid)
549              if transB
550              else torch.rand(b, k, n, dtype=dtype, device=deviceid)
551          )
552          matA = matA.transpose(1, 2) if transA else matA
553          matB = matB.transpose(1, 2) if transB else matB
554          return matA, matB

557  def _process_single_offline_gemm(untuned_gemm_line: str, gpu_id: int) -> None:
558      r"""Process a single untuned GEMM."""
559  
560      deviceid = "cuda:" + str(gpu_id)
561  
562      dtype_dict = {
563          "float": torch.float32,
564          "tf32": torch.float32,
565          "double": torch.float64,
566          "BFloat16": torch.bfloat16,
567          "Half": torch.half,
568          "c10::complex<double>": torch.complex128,
569          "c10::complex<float>": torch.complex64,
570          "Float8_e4m3fn": torch.float8_e4m3fn,
571          "Float8_e5m2": torch.float8_e5m2,
572          "Float8_e4m3fnuz": torch.float8_e4m3fnuz,
573          "Float8_e5m2fnuz": torch.float8_e5m2fnuz,
574      }
575  
576      untuned_gemm = untuned_gemm_line.strip().split(",")[:]
577  
578      underscore_count = untuned_gemm[0].count("_")
579  
580      # Initialize dtype to make linter happy
581      dtype = None
582      dtypeA = None
583      dtypeB = None
584      dtypeC = None
585  
586      # Extract BLAS parameters
587      if underscore_count == 2:
588          [op_sig, data_type, layout] = untuned_gemm[0].split("_")
589          transB = layout[0] == "T"
590          transA = layout[1] == "T"
591          dtype = dtype_dict.get(data_type)
592          if data_type == "tf32":
593              # User must still set HIPBLASLT_ALLOW_TF32=1
594              torch.backends.cuda.matmul.allow_tf32 = True
595          else:
596              torch.backends.cuda.matmul.allow_tf32 = False
597  
598      else:  # ScaledGEMM
599          count = untuned_gemm[0].count("_")
600          assert count in [6, 7]
601          untuned_gemm_temp = untuned_gemm[0].split("_")
602          # dtypeC = might not be FP8 type, keep track
603          # of the the number of underscores
604          op_sig = untuned_gemm_temp[0]
605          data_typeA = untuned_gemm_temp[1] + "_" + untuned_gemm_temp[2]
606          data_typeB = untuned_gemm_temp[3] + "_" + untuned_gemm_temp[4]
607          if count == 7:
608              data_typeC = untuned_gemm_temp[5] + "_" + untuned_gemm_temp[6]
609          else:
610              data_typeC = untuned_gemm_temp[5]
611          transB = untuned_gemm_temp[count][0] == "T"
612          transA = untuned_gemm_temp[count][1] == "T"
613          dtypeA = dtype_dict.get(data_typeA)
614          dtypeB = dtype_dict.get(data_typeB)
615          dtypeC = dtype_dict.get(data_typeC)
616  
617      untuned_gemm_temp = untuned_gemm[1].split("_")
618      [n, m, k] = [int(g) for g in untuned_gemm_temp[1:4]]
619      if op_sig == "GemmStridedBatchedTunableOp":
620          assert untuned_gemm_temp[6] == "ld"
621          [ldb, lda, ldc] = [int(g) for g in untuned_gemm_temp[7:10]]
622      else:
623          assert untuned_gemm_temp[4] == "ld"
624          [ldb, lda, ldc] = [int(g) for g in untuned_gemm_temp[5:8]]
625  
626      # Detect subMatrix case
627      if all(item in [n, m, k] for item in [lda, ldb, ldc]):
628          subMatrix = False
629      else:
630          subMatrix = True
631  
632      if op_sig == "GemmTunableOp":
633          # Warnings for unsupported cases:
634          if m == 1 or n == 1 or k == 1:
635              if (not transA) and (not transB):
636                  pass  # case is supported
637              elif transA and n == 1:
638                  pass  # case is supported
639              else:
640                  warnings.warn(
641                      "Offline tuning is not supported for this GEMM. Use online tuning instead. "
642                      + f"Skipped tuning for: {untuned_gemm[1]}"
643                  )
644                  return
645  
646          # Resolve linter issue
647          if dtype is None or not isinstance(dtype, torch.dtype):
648              raise TypeError(f"dtype must be a torch.dtype, but got {dtype}")
649  
650          matA, matB = _create_matrices(
651              m, n, k, lda, ldb, ldc, transA, transB, dtype, deviceid, subMatrix=subMatrix
652          )
653          torch.mm(matA, matB)
654  
655      elif op_sig == "GemmStridedBatchedTunableOp":
656          # Warnings for unsupported cases:
657          if m == 1 or n == 1 or k == 1:
658              warnings.warn(
659                  "Offline tuning is not support for this GEMM. Use online tuning instead. "
660                  + f"Skipped tuning for: {untuned_gemm[1]}"
661              )
662              return
663  
664          [b] = [int(g) for g in untuned_gemm_temp[5:6]]
665  
666          # Resolve linter issue
667          if dtype is None or not isinstance(dtype, torch.dtype):
668              raise TypeError(f"dtype must be a torch.dtype, but got {dtype}")
669  
670          matA, matB = _create_batch_matrices(
671              m,
672              n,
673              k,
674              b,
675              lda,
676              ldb,
677              ldc,
678              transA,
679              transB,
680              dtype,
681              deviceid,
682              subMatrix=subMatrix,
683          )
684          torch.bmm(matA, matB)
685      elif op_sig == "ScaledGemmTunableOp":
686          # Only combination supported by PyTorch
687          assert transB is True
688          assert transA is False
689  
690          # Resolve linter issue
691          if dtypeA is None or not isinstance(dtypeA, torch.dtype):
692              raise TypeError(f"dtype must be a torch.dtype, but got {dtypeA}")
693  
694          matA, matB = _create_matrices(
695              m,
696              n,
697              k,
698              lda,
699              ldb,
700              ldc,
701              transA,
702              transB,
703              dtypeA,
704              deviceid,
705              dtypeB=dtypeB,
706              randn=False,
707              subMatrix=subMatrix,
708          )
709  
710          assert untuned_gemm_temp[8] == "rw"
711          if untuned_gemm_temp[9] == "1":
712              rowwise = True
713          else:
714              rowwise = False
715          if rowwise:
716              scaleA = (
717                  torch.ones((1, m), device=deviceid)
718                  if transA
719                  else torch.ones((m, 1), device=deviceid)
720              )
721              scaleB = (
722                  torch.ones((1, n), device=deviceid)
723                  if transB
724                  else torch.ones((n, 1), device=deviceid)
725              )
726          else:
727              scaleA = torch.tensor(0.8, device=deviceid)
728              scaleB = torch.tensor(0.9, device=deviceid)
729  
730          assert untuned_gemm_temp[10] == "bias"
731          if untuned_gemm_temp[11] == "None":  # no bias vector
732              torch._scaled_mm(
733                  matA, matB, scale_a=scaleA, scale_b=scaleB, out_dtype=dtypeC
734              )
735          else:  # bias vector present
736              fillbias = 0.10
737              bias_dtype = dtype_dict.get(untuned_gemm_temp[11])
738              bias = (
739                  torch.full((n,), fillbias, dtype=bias_dtype, device=deviceid)
740                  if transB
741                  else torch.full((m,), fillbias, dtype=bias_dtype, device=deviceid)
742              )
743              torch._scaled_mm(
744                  matA, matB, scale_a=scaleA, scale_b=scaleB, out_dtype=dtypeC, bias=bias
745              )
746  
747      elif op_sig == "GemmAndBiasTunableOp":
748          # y = x*A^T + b
749          assert transA != transB
750  
751          # Resolve linter issue
752          if dtype is None or not isinstance(dtype, torch.dtype):
753              raise TypeError(f"dtype must be a torch.dtype, but got {dtype}")
754  
755          bias = torch.rand(n, dtype=dtype, device=deviceid)
756  
757          X, matA = _create_matrices(
758              m, n, k, lda, ldb, ldc, transA, transB, dtype, deviceid, subMatrix=subMatrix
759          )
760          matA = matA.t()
761          torch.nn.functional.linear(X, matA, bias)
762      else:
763          warnings.warn(f"error: unknown op {op_sig}")

766  def _check_tuning_assertions() -> None:
767      r"""Helper function for multi-GPU tuning case. Need to check that TunableOp feature 
768      is enabled and that tuning is enabled. 
769      """
770  
771      if is_enabled() is False:
772          warnings.warn("TunableOp was disabled. Trying to enable now.")
773          enable(True)
774      assert is_enabled() is True
775      assert tuning_is_enabled() is True
776      assert record_untuned_is_enabled() is False

779  def mgpu_tune_gemm_in_file(filename_pattern: str, num_gpus: int) -> None:
780      r"""Process one or more files and distribute work over one or more GPUs."""
781      unique_gemm_entries = _gather_unique_untuned_gemm_from_files(filename_pattern)
782  
783      total_gpus = torch.cuda.device_count()
784  
785      assert 1 <= num_gpus <= total_gpus
786  
787      mp_context = mp.get_context("spawn")
788  
789      futures = []  # empty list to hold futures
790      flush_results = []  # empty list to hold futures
791  
792      # GEMM are assigned to GPUs in a round robin manner
793      h = 0
794      with concurrent.futures.ProcessPoolExecutor(
795          max_workers=num_gpus,
796          mp_context=mp_context,
797          initializer=_check_tuning_assertions,
798      ) as executor:
799          # The workers are a separate process. TunableOp will be
800          # enabled in the child processes if PYTORCH_TUNABLEOP_ENABLED=1
801          # In the initializer, we also try to enable TunableOP if th
802          # environment variable was NOT set.
803  
804          for line in unique_gemm_entries:
805              future = executor.submit(_process_single_offline_gemm, line, h)
806              futures.append(future)
807              h = (h + 1) % num_gpus
808  
809          for future in concurrent.futures.as_completed(futures):
810              future.result()
811  
812          for g in range(num_gpus):
813              flush_result = executor.submit(write_file)
814              flush_results.append(flush_result)
815  
816          for flush_result in concurrent.futures.as_completed(flush_results):
817              flush_result.result()
818  
819      torch.cuda.synchronize()
820  
821      _gather_tunableop_results()
