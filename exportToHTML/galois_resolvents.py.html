galois_resolvents.py 
1    r""" 
2    Galois resolvents 
3     
4    Each of the functions in ``sympy.polys.numberfields.galoisgroups`` that 
5    computes Galois groups for a particular degree $n$ uses resolvents. Given the 
6    polynomial $T$ whose Galois group is to be computed, a resolvent is a 
7    polynomial $R$ whose roots are defined as functions of the roots of $T$. 
8     
9    One way to compute the coefficients of $R$ is by approximating the roots of $T$ 
10   to sufficient precision. This module defines a :py:class:`~.Resolvent` class 
11   that handles this job, determining the necessary precision, and computing $R$. 
12    
13   In some cases, the coefficients of $R$ are symmetric in the roots of $T$, 
14   meaning they are equal to fixed functions of the coefficients of $T$. Therefore 
15   another approach is to compute these functions once and for all, and record 
16   them in a lookup table. This module defines code that can compute such tables. 
17   The tables for polynomials $T$ of degrees 4 through 6, produced by this code, 
18   are recorded in the resolvent_lookup.py module. 
19    
20   """
21

47   class Resolvent:
48       r""" 
49       If $G$ is a subgroup of the symmetric group $S_n$, 
50       $F$ a multivariate polynomial in $\mathbb{Z}[X_1, \ldots, X_n]$, 
51       $H$ the stabilizer of $F$ in $G$ (i.e. the permutations $\sigma$ such that 
52       $F(X_{\sigma(1)}, \ldots, X_{\sigma(n)}) = F(X_1, \ldots, X_n)$), and $s$ 
53       a set of left coset representatives of $H$ in $G$, then the resolvent 
54       polynomial $R(Y)$ is the product over $\sigma \in s$ of 
55       $Y - F(X_{\sigma(1)}, \ldots, X_{\sigma(n)})$. 
56    
57       For example, consider the resolvent for the form 
58       $$F = X_0 X_2 + X_1 X_3$$ 
59       and the group $G = S_4$. In this case, the stabilizer $H$ is the dihedral 
60       group $D4 = < (0123), (02) >$, and a set of representatives of $G/H$ is 
61       $\{I, (01), (03)\}$. The resolvent can be constructed as follows: 
62    
63       >>> from sympy.combinatorics.permutations import Permutation 
64       >>> from sympy.core.symbol import symbols 
65       >>> from sympy.polys.numberfields.galoisgroups import Resolvent 
66       >>> X = symbols('X0 X1 X2 X3') 
67       >>> F = X[0]*X[2] + X[1]*X[3] 
68       >>> s = [Permutation([0, 1, 2, 3]), Permutation([1, 0, 2, 3]), 
69       ... Permutation([3, 1, 2, 0])] 
70       >>> R = Resolvent(F, X, s) 
71    
72       This resolvent has three roots, which are the conjugates of ``F`` under the 
73       three permutations in ``s``: 
74    
75       >>> R.root_lambdas[0](*X) 
76       X0*X2 + X1*X3 
77       >>> R.root_lambdas[1](*X) 
78       X0*X3 + X1*X2 
79       >>> R.root_lambdas[2](*X) 
80       X0*X1 + X2*X3 
81    
82       Resolvents are useful for computing Galois groups. Given a polynomial $T$ 
83       of degree $n$, we will use a resolvent $R$ where $Gal(T) \leq G \leq S_n$. 
84       We will then want to substitute the roots of $T$ for the variables $X_i$ 
85       in $R$, and study things like the discriminant of $R$, and the way $R$ 
86       factors over $\mathbb{Q}$. 
87    
88       From the symmetry in $R$'s construction, and since $Gal(T) \leq G$, we know 
89       from Galois theory that the coefficients of $R$ must lie in $\mathbb{Z}$. 
90       This allows us to compute the coefficients of $R$ by approximating the 
91       roots of $T$ to sufficient precision, plugging these values in for the 
92       variables $X_i$ in the coefficient expressions of $R$, and then simply 
93       rounding to the nearest integer. 
94    
95       In order to determine a sufficient precision for the roots of $T$, this 
96       ``Resolvent`` class imposes certain requirements on the form ``F``. It 
97       could be possible to design a different ``Resolvent`` class, that made 
98       different precision estimates, and different assumptions about ``F``. 
99    
100      ``F`` must be homogeneous, and all terms must have unit coefficient. 
101      Furthermore, if $r$ is the number of terms in ``F``, and $t$ the total 
102      degree, and if $m$ is the number of conjugates of ``F``, i.e. the number 
103      of permutations in ``s``, then we require that $m < r 2^t$. Again, it is 
104      not impossible to work with forms ``F`` that violate these assumptions, but 
105      this ``Resolvent`` class requires them. 
106   
107      Since determining the integer coefficients of the resolvent for a given 
108      polynomial $T$ is one of the main problems this class solves, we take some 
109      time to explain the precision bounds it uses. 
110   
111      The general problem is: 
112      Given a multivariate polynomial $P \in \mathbb{Z}[X_1, \ldots, X_n]$, and a 
113      bound $M \in \mathbb{R}_+$, compute an $\varepsilon > 0$ such that for any 
114      complex numbers $a_1, \ldots, a_n$ with $|a_i| < M$, if the $a_i$ are 
115      approximated to within an accuracy of $\varepsilon$ by $b_i$, that is, 
116      $|a_i - b_i| < \varepsilon$ for $i = 1, \ldots, n$, then 
117      $|P(a_1, \ldots, a_n) - P(b_1, \ldots, b_n)| < 1/2$. In other words, if it 
118      is known that $P(a_1, \ldots, a_n) = c$ for some $c \in \mathbb{Z}$, then 
119      $P(b_1, \ldots, b_n)$ can be rounded to the nearest integer in order to 
120      determine $c$. 
121   
122      To derive our error bound, consider the monomial $xyz$. Defining 
123      $d_i = b_i - a_i$, our error is 
124      $|(a_1 + d_1)(a_2 + d_2)(a_3 + d_3) - a_1 a_2 a_3|$, which is bounded 
125      above by $|(M + \varepsilon)^3 - M^3|$. Passing to a general monomial of 
126      total degree $t$, this expression is bounded by 
127      $M^{t-1}\varepsilon(t + 2^t\varepsilon/M)$ provided $\varepsilon < M$, 
128      and by $(t+1)M^{t-1}\varepsilon$ provided $\varepsilon < M/2^t$. 
129      But since our goal is to make the error less than $1/2$, we will choose 
130      $\varepsilon < 1/(2(t+1)M^{t-1})$, which implies the condition that 
131      $\varepsilon < M/2^t$, as long as $M \geq 2$. 
132   
133      Passing from the general monomial to the general polynomial is easy, by 
134      scaling and summing error bounds. 
135   
136      In our specific case, we are given a homogeneous polynomial $F$ of 
137      $r$ terms and total degree $t$, all of whose coefficients are $\pm 1$. We 
138      are given the $m$ permutations that make the conjugates of $F$, and 
139      we want to bound the error in the coefficients of the monic polynomial 
140      $R(Y)$ having $F$ and its conjugates as roots (i.e. the resolvent). 
141   
142      For $j$ from $1$ to $m$, the coefficient of $Y^{m-j}$ in $R(Y)$ is the 
143      $j$th elementary symmetric polynomial in the conjugates of $F$. This sums 
144      the products of these conjugates, taken $j$ at a time, in all possible 
145      combinations. There are $\binom{m}{j}$ such combinations, and each product 
146      of $j$ conjugates of $F$ expands to a sum of $r^j$ terms, each of unit 
147      coefficient, and total degree $jt$. An error bound for the $j$th coeff of 
148      $R$ is therefore 
149      $$\binom{m}{j} r^j (jt + 1) M^{jt - 1} \varepsilon$$ 
150      When our goal is to evaluate all the coefficients of $R$, we will want to 
151      use the maximum of these error bounds. It is clear that this bound is 
152      strictly increasing for $j$ up to the ceiling of $m/2$. After that point, 
153      the first factor $\binom{m}{j}$ begins to decrease, while the others 
154      continue to increase. However, the binomial coefficient never falls by more 
155      than a factor of $1/m$ at a time, so our assumptions that $M \geq 2$ and 
156      $m < r 2^t$ are enough to tell us that the constant coefficient of $R$, 
157      i.e. that where $j = m$, has the largest error bound. Therefore we can use 
158      $$r^m (mt + 1) M^{mt - 1} \varepsilon$$ 
159      as our error bound for all the coefficients. 
160   
161      Note that this bound is also (more than) adequate to determine whether any 
162      of the roots of $R$ is an integer. Each of these roots is a single 
163      conjugate of $F$, which contains less error than the trace, i.e. the 
164      coefficient of $Y^{m - 1}$. By rounding the roots of $R$ to the nearest 
165      integers, we therefore get all the candidates for integer roots of $R$. By 
166      plugging these candidates into $R$, we can check whether any of them 
167      actually is a root. 
168   
169      Note: We take the definition of resolvent from Cohen, but the error bound 
170      is ours. 
171   
172      References 
173      ========== 
174   
175      .. [1] Cohen, H. *A Course in Computational Algebraic Number Theory*. 
176         (Def 6.3.2) 
177   
178      """
179  
180      def __init__(self, F, X, s):
181          r""" 
182          Parameters 
183          ========== 
184   
185          F : :py:class:`~.Expr` 
186              polynomial in the symbols in *X* 
187          X : list of :py:class:`~.Symbol` 
188          s : list of :py:class:`~.Permutation` 
189              representing the cosets of the stabilizer of *F* in 
190              some subgroup $G$ of $S_n$, where $n$ is the length of *X*. 
191          """
192          self.F = F
193          self.X = X
194          self.s = s
195  
196          # Number of conjugates:
197          self.m = len(s)
198          # Total degree of F (computed below):
199          self.t = None
200          # Number of terms in F (computed below):
201          self.r = 0
202  
203          for monom, coeff in Poly(F).terms():
204              if abs(coeff) != 1:
205                  raise ResolventException('Resolvent class expects forms with unit coeffs')
206              t = sum(monom)
207              if t != self.t and self.t is not None:
208                  raise ResolventException('Resolvent class expects homogeneous forms')
209              self.t = t
210              self.r += 1
211  
212          m, t, r = self.m, self.t, self.r
213          if not m < r * 2**t:
214              raise ResolventException('Resolvent class expects m < r*2^t')
215          M = symbols('M')
216          # Precision sufficient for computing the coeffs of the resolvent:
217          self.coeff_prec_func = Poly(r**m*(m*t + 1)*M**(m*t - 1))
218          # Precision sufficient for checking whether any of the roots of the
219          # resolvent are integers:
220          self.root_prec_func = Poly(r*(t + 1)*M**(t - 1))
221  
222          # The conjugates of F are the roots of the resolvent.
223          # For evaluating these to required numerical precisions, we need
224          # lambdified versions.
225          # Note: for a given permutation sigma, the conjugate (sigma F) is
226          # equivalent to lambda [sigma^(-1) X]: F.
227          self.root_lambdas = [
228              lambdify((~s[j])(X), F)
229              for j in range(self.m)
230          ]
231  
232          # For evaluating the coeffs, we'll also need lambdified versions of
233          # the elementary symmetric functions for degree m.
234          Y = symbols('Y')
235          R = symbols(' '.join(f'R{i}' for i in range(m)))
236          f = 1
237          for r in R:
238              f *= (Y - r)
239          C = Poly(f, Y).coeffs()
240          self.esf_lambdas = [lambdify(R, c) for c in C]
241  
242      def get_prec(self, M, target='coeffs'):
243          r""" 
244          For a given upper bound *M* on the magnitude of the complex numbers to 
245          be plugged in for this resolvent's symbols, compute a sufficient 
246          precision for evaluating those complex numbers, such that the 
247          coefficients, or the integer roots, of the resolvent can be determined. 
248   
249          Parameters 
250          ========== 
251   
252          M : real number 
253              Upper bound on magnitude of the complex numbers to be plugged in. 
254   
255          target : str, 'coeffs' or 'roots', default='coeffs' 
256              Name the task for which a sufficient precision is desired. 
257              This is either determining the coefficients of the resolvent 
258              ('coeffs') or determining its possible integer roots ('roots'). 
259              The latter may require significantly lower precision. 
260   
261          Returns 
262          ======= 
263   
264          int $m$ 
265              such that $2^{-m}$ is a sufficient upper bound on the 
266              error in approximating the complex numbers to be plugged in. 
267   
268          """
269          # As explained in the docstring for this class, our precision estimates
270          # require that M be at least 2.
271          M = max(M, 2)
272          f = self.coeff_prec_func if target == 'coeffs' else self.root_prec_func
273          r, _, _, _ = evalf(2*f(M), 1, {})
274          return fastlog(r) + 1
275  
276      def approximate_roots_of_poly(self, T, target='coeffs'):
277          """ 
278          Approximate the roots of a given polynomial *T* to sufficient precision 
279          in order to evaluate this resolvent's coefficients, or determine 
280          whether the resolvent has an integer root. 
281   
282          Parameters 
283          ========== 
284   
285          T : :py:class:`~.Poly` 
286   
287          target : str, 'coeffs' or 'roots', default='coeffs' 
288              Set the approximation precision to be sufficient for the desired 
289              task, which is either determining the coefficients of the resolvent 
290              ('coeffs') or determining its possible integer roots ('roots'). 
291              The latter may require significantly lower precision. 
292   
293          Returns 
294          ======= 
295   
296          list of elements of :ref:`CC` 
297   
298          """
299          ctx = MPContext()
300          # Because sympy.polys.polyroots._integer_basis() is called when a CRootOf
301          # is formed, we proactively extract the integer basis now. This means that
302          # when we call T.all_roots(), every root will be a CRootOf, not a Mul
303          # of Integer*CRootOf.
304          coeff, T = preprocess_roots(T)
305          coeff = ctx.mpf(str(coeff))
306  
307          scaled_roots = T.all_roots(radicals=False)
308  
309          # Since we're going to be approximating the roots of T anyway, we can
310          # get a good upper bound on the magnitude of the roots by starting with
311          # a very low precision approx.
312          approx0 = [coeff * quad_to_mpmath(_evalf_with_bounded_error(r, m=0)) for r in scaled_roots]
313          # Here we add 1 to account for the possible error in our initial approximation.
314          M = max(abs(b) for b in approx0) + 1
315          m = self.get_prec(M, target=target)
316          n = fastlog(M._mpf_) + 1
317          p = m + n + 1
318          ctx.prec = p
319          d = prec_to_dps(p)
320  
321          approx1 = [r.eval_approx(d, return_mpmath=True) for r in scaled_roots]
322          approx1 = [coeff*ctx.mpc(r) for r in approx1]
323  
324          return approx1
325  
326      @staticmethod
327      def round_mpf(a):
328          if isinstance(a, int):
329              return a
330          # If we use python's built-in `round()`, we lose precision.
331          # If we use `ZZ` directly, we may add or subtract 1.
332          #
333          # XXX: We have to convert to int before converting to ZZ because
334          # flint.fmpz cannot convert a mpmath mpf.
335          return ZZ(int(a.context.nint(a)))

337      def round_roots_to_integers_for_poly(self, T):
338          """ 
339          For a given polynomial *T*, round the roots of this resolvent to the 
340          nearest integers. 
341   
342          Explanation 
343          =========== 
344   
345          None of the integers returned by this method is guaranteed to be a 
346          root of the resolvent; however, if the resolvent has any integer roots 
347          (for the given polynomial *T*), then they must be among these. 
348   
349          If the coefficients of the resolvent are also desired, then this method 
350          should not be used. Instead, use the ``eval_for_poly`` method. This 
351          method may be significantly faster than ``eval_for_poly``. 
352   
353          Parameters 
354          ========== 
355   
356          T : :py:class:`~.Poly` 
357   
358          Returns 
359          ======= 
360   
361          dict 
362              Keys are the indices of those permutations in ``self.s`` such that 
363              the corresponding root did round to a rational integer. 
364   
365              Values are :ref:`ZZ`. 
366   
367   
368          """
369          approx_roots_of_T = self.approximate_roots_of_poly(T, target='roots')
370          approx_roots_of_self = [r(*approx_roots_of_T) for r in self.root_lambdas]
371          return {
372              i: self.round_mpf(r.real)
373              for i, r in enumerate(approx_roots_of_self)
374              if self.round_mpf(r.imag) == 0
375          }

377      def eval_for_poly(self, T, find_integer_root=False):
378          r""" 
379          Compute the integer values of the coefficients of this resolvent, when 
380          plugging in the roots of a given polynomial. 
381   
382          Parameters 
383          ========== 
384   
385          T : :py:class:`~.Poly` 
386   
387          find_integer_root : ``bool``, default ``False`` 
388              If ``True``, then also determine whether the resolvent has an 
389              integer root, and return the first one found, along with its 
390              index, i.e. the index of the permutation ``self.s[i]`` it 
391              corresponds to. 
392   
393          Returns 
394          ======= 
395   
396          Tuple ``(R, a, i)`` 
397   
398              ``R`` is this resolvent as a dense univariate polynomial over 
399              :ref:`ZZ`, i.e. a list of :ref:`ZZ`. 
400   
401              If *find_integer_root* was ``True``, then ``a`` and ``i`` are the 
402              first integer root found, and its index, if one exists. 
403              Otherwise ``a`` and ``i`` are both ``None``. 
404   
405          """
406          approx_roots_of_T = self.approximate_roots_of_poly(T, target='coeffs')
407          approx_roots_of_self = [r(*approx_roots_of_T) for r in self.root_lambdas]
408          approx_coeffs_of_self = [c(*approx_roots_of_self) for c in self.esf_lambdas]
409  
410          R = []
411          for c in approx_coeffs_of_self:
412              if self.round_mpf(c.imag) != 0:
413                  # If precision was enough, this should never happen.
414                  raise ResolventException(f"Got non-integer coeff for resolvent: {c}")
415              R.append(self.round_mpf(c.real))
416  
417          a0, i0 = None, None
418  
419          if find_integer_root:
420              for i, r in enumerate(approx_roots_of_self):
421                  if self.round_mpf(r.imag) != 0:
422                      continue
423                  if not dup_eval(R, (a := self.round_mpf(r.real)), ZZ):
424                      a0, i0 = a, i
425                      break
426  
427          return R, a0, i0

430  def wrap(text, width=80):
431      """Line wrap a polynomial expression. """
432      out = ''
433      col = 0
434      for c in text:
435          if c == ' ' and col > width:
436              c, col = '\n', 0
437          else:
438              col += 1
439          out += c
440      return out

443  def s_vars(n):
444      """Form the symbols s1, s2, ..., sn to stand for elem. symm. polys. """
445      return symbols([f's{i + 1}' for i in range(n)])

448  def sparse_symmetrize_resolvent_coeffs(F, X, s, verbose=False):
449      """ 
450      Compute the coefficients of a resolvent as functions of the coefficients of 
451      the associated polynomial. 
452   
453      F must be a sparse polynomial. 
454      """
455      import time, sys
456      # Roots of resolvent as multivariate forms over vars X:
457      root_forms = [
458          F.compose(list(zip(X, sigma(X))))
459          for sigma in s
460      ]
461  
462      # Coeffs of resolvent (besides lead coeff of 1) as symmetric forms over vars X:
463      Y = [Dummy(f'Y{i}') for i in range(len(s))]
464      coeff_forms = []
465      for i in range(1, len(s) + 1):
466          if verbose:
467              print('----')
468              print(f'Computing symmetric poly of degree {i}...')
469              sys.stdout.flush()
470          t0 = time.time()
471          G = symmetric_poly(i, *Y)
472          t1 = time.time()
473          if verbose:
474              print(f'took {t1 - t0} seconds')
475              print('lambdifying...')
476              sys.stdout.flush()
477          t0 = time.time()
478          C = lambdify(Y, (-1)**i*G)
479          t1 = time.time()
480          if verbose:
481              print(f'took {t1 - t0} seconds')
482              sys.stdout.flush()
483          coeff_forms.append(C)
484  
485      coeffs = []
486      for i, f in enumerate(coeff_forms):
487          if verbose:
488              print('----')
489              print(f'Plugging root forms into elem symm poly {i+1}...')
490              sys.stdout.flush()
491          t0 = time.time()
492          g = f(*root_forms)
493          t1 = time.time()
494          coeffs.append(g)
495          if verbose:
496              print(f'took {t1 - t0} seconds')
497              sys.stdout.flush()
498  
499      # Now symmetrize these coeffs. This means recasting them as polynomials in
500      # the elementary symmetric polys over X.
501      symmetrized = []
502      symmetrization_times = []
503      ss = s_vars(len(X))
504      for i, A in list(enumerate(coeffs)):
505          if verbose:
506              print('-----')
507              print(f'Coeff {i+1}...')
508              sys.stdout.flush()
509          t0 = time.time()
510          B, rem, _ = A.symmetrize()
511          t1 = time.time()
512          if rem != 0:
513              msg = f"Got nonzero remainder {rem} for resolvent (F, X, s) = ({F}, {X}, {s})"
514              raise ResolventException(msg)
515          B_str = str(B.as_expr(*ss))
516          symmetrized.append(B_str)
517          symmetrization_times.append(t1 - t0)
518          if verbose:
519              print(wrap(B_str))
520              print(f'took {t1 - t0} seconds')
521              sys.stdout.flush()
522  
523      return symmetrized, symmetrization_times

526  def define_resolvents():
527      """Define all the resolvents for polys T of degree 4 through 6. """
528      from sympy.combinatorics.galois import PGL2F5
529      from sympy.combinatorics.permutations import Permutation
530  
531      R4, X4 = xring("X0,X1,X2,X3", ZZ, lex)
532      X = X4
533  
534      # The one resolvent used in `_galois_group_degree_4_lookup()`:
535      F40 = X[0]*X[1]**2 + X[1]*X[2]**2 + X[2]*X[3]**2 + X[3]*X[0]**2
536      s40 = [
537          Permutation(3),
538          Permutation(3)(0, 1),
539          Permutation(3)(0, 2),
540          Permutation(3)(0, 3),
541          Permutation(3)(1, 2),
542          Permutation(3)(2, 3),
543      ]
544  
545      # First resolvent used in `_galois_group_degree_4_root_approx()`:
546      F41 = X[0]*X[2] + X[1]*X[3]
547      s41 = [
548          Permutation(3),
549          Permutation(3)(0, 1),
550          Permutation(3)(0, 3)
551      ]
552  
553      R5, X5 = xring("X0,X1,X2,X3,X4", ZZ, lex)
554      X = X5
555  
556      # First resolvent used in `_galois_group_degree_5_hybrid()`,
557      # and only one used in `_galois_group_degree_5_lookup_ext_factor()`:
558      F51 = (  X[0]**2*(X[1]*X[4] + X[2]*X[3])
559             + X[1]**2*(X[2]*X[0] + X[3]*X[4])
560             + X[2]**2*(X[3]*X[1] + X[4]*X[0])
561             + X[3]**2*(X[4]*X[2] + X[0]*X[1])
562             + X[4]**2*(X[0]*X[3] + X[1]*X[2]))
563      s51 = [
564          Permutation(4),
565          Permutation(4)(0, 1),
566          Permutation(4)(0, 2),
567          Permutation(4)(0, 3),
568          Permutation(4)(0, 4),
569          Permutation(4)(1, 4)
570      ]
571  
572      R6, X6 = xring("X0,X1,X2,X3,X4,X5", ZZ, lex)
573      X = X6
574  
575      # First resolvent used in `_galois_group_degree_6_lookup()`:
576      H = PGL2F5()
577      term0 = X[0]**2*X[5]**2*(X[1]*X[4] + X[2]*X[3])
578      terms = {term0.compose(list(zip(X, s(X)))) for s in H.elements}
579      F61 = sum(terms)
580      s61 = [Permutation(5)] + [Permutation(5)(0, n) for n in range(1, 6)]
581  
582      # Second resolvent used in `_galois_group_degree_6_lookup()`:
583      F62 = X[0]*X[1]*X[2] + X[3]*X[4]*X[5]
584      s62 = [Permutation(5)] + [
585          Permutation(5)(i, j + 3) for i in range(3) for j in range(3)
586      ]
587  
588      return {
589          (4, 0): (F40, X4, s40),
590          (4, 1): (F41, X4, s41),
591          (5, 1): (F51, X5, s51),
592          (6, 1): (F61, X6, s61),
593          (6, 2): (F62, X6, s62),
594      }

597  def generate_lambda_lookup(verbose=False, trial_run=False):
598      """ 
599      Generate the whole lookup table of coeff lambdas, for all resolvents. 
600      """
601      jobs = define_resolvents()
602      lambda_lists = {}
603      total_time = 0
604      time_for_61 = 0
605      time_for_61_last = 0
606      for k, (F, X, s) in jobs.items():
607          symmetrized, times = sparse_symmetrize_resolvent_coeffs(F, X, s, verbose=verbose)
608  
609          total_time += sum(times)
610          if k == (6, 1):
611              time_for_61 = sum(times)
612              time_for_61_last = times[-1]
613  
614          sv = s_vars(len(X))
615          head = f'lambda {", ".join(str(v) for v in sv)}:'
616          lambda_lists[k] = ',\n        '.join([
617              f'{head} ({wrap(f)})'
618              for f in symmetrized
619          ])
620  
621          if trial_run:
622              break
623  
624      table = (
625           "# This table was generated by a call to\n"
626           "# `sympy.polys.numberfields.galois_resolvents.generate_lambda_lookup()`.\n"
627          f"# The entire job took {total_time:.2f}s.\n"
628          f"# Of this, Case (6, 1) took {time_for_61:.2f}s.\n"
629          f"# The final polynomial of Case (6, 1) alone took {time_for_61_last:.2f}s.\n"
630           "resolvent_coeff_lambdas = {\n")
631  
632      for k, L in lambda_lists.items():
633          table += f"    {k}: [\n"
634          table +=  "        " + L + '\n'
635          table +=  "    ],\n"
636      table += "}\n"
637      return table

640  def get_resolvent_by_lookup(T, number):
641      """ 
642      Use the lookup table, to return a resolvent (as dup) for a given 
643      polynomial *T*. 
644   
645      Parameters 
646      ========== 
647   
648      T : Poly 
649          The polynomial whose resolvent is needed 
650   
651      number : int 
652          For some degrees, there are multiple resolvents. 
653          Use this to indicate which one you want. 
654   
655      Returns 
656      ======= 
657   
658      dup 
659   
660      """
661      from sympy.polys.numberfields.resolvent_lookup import resolvent_coeff_lambdas
662      degree = T.degree()
663      L = resolvent_coeff_lambdas[(degree, number)]
664      T_coeffs = T.rep.to_list()[1:]
665      return [ZZ(1)] + [c(*T_coeffs) for c in L]
