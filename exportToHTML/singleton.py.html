24   class SingletonRegistry(Registry):
25       """ 
26       The registry for the singleton classes (accessible as ``S``). 
27    
28       Explanation 
29       =========== 
30    
31       This class serves as two separate things. 
32    
33       The first thing it is is the ``SingletonRegistry``. Several classes in 
34       SymPy appear so often that they are singletonized, that is, using some 
35       metaprogramming they are made so that they can only be instantiated once 
36       (see the :class:`sympy.core.singleton.Singleton` class for details). For 
37       instance, every time you create ``Integer(0)``, this will return the same 
38       instance, :class:`sympy.core.numbers.Zero`. All singleton instances are 
39       attributes of the ``S`` object, so ``Integer(0)`` can also be accessed as 
40       ``S.Zero``. 
41    
42       Singletonization offers two advantages: it saves memory, and it allows 
43       fast comparison. It saves memory because no matter how many times the 
44       singletonized objects appear in expressions in memory, they all point to 
45       the same single instance in memory. The fast comparison comes from the 
46       fact that you can use ``is`` to compare exact instances in Python 
47       (usually, you need to use ``==`` to compare things). ``is`` compares 
48       objects by memory address, and is very fast. 
49    
50       Examples 
51       ======== 
52    
53       >>> from sympy import S, Integer 
54       >>> a = Integer(0) 
55       >>> a is S.Zero 
56       True 
57    
58       For the most part, the fact that certain objects are singletonized is an 
59       implementation detail that users should not need to worry about. In SymPy 
60       library code, ``is`` comparison is often used for performance purposes 
61       The primary advantage of ``S`` for end users is the convenient access to 
62       certain instances that are otherwise difficult to type, like ``S.Half`` 
63       (instead of ``Rational(1, 2)``). 
64    
65       When using ``is`` comparison, make sure the argument is sympified. For 
66       instance, 
67    
68       >>> x = 0 
69       >>> x is S.Zero 
70       False 
71    
72       This problem is not an issue when using ``==``, which is recommended for 
73       most use-cases: 
74    
75       >>> 0 == S.Zero 
76       True 
77    
78       The second thing ``S`` is is a shortcut for 
79       :func:`sympy.core.sympify.sympify`. :func:`sympy.core.sympify.sympify` is 
80       the function that converts Python objects such as ``int(1)`` into SymPy 
81       objects such as ``Integer(1)``. It also converts the string form of an 
82       expression into a SymPy expression, like ``sympify("x**2")`` -> 
83       ``Symbol("x")**2``. ``S(1)`` is the same thing as ``sympify(1)`` 
84       (basically, ``S.__call__`` has been defined to call ``sympify``). 
85    
86       This is for convenience, since ``S`` is a single letter. It's mostly 
87       useful for defining rational numbers. Consider an expression like ``x + 
88       1/2``. If you enter this directly in Python, it will evaluate the ``1/2`` 
89       and give ``0.5``, because both arguments are ints (see also 
90       :ref:`tutorial-gotchas-final-notes`). However, in SymPy, you usually want 
91       the quotient of two integers to give an exact rational number. The way 
92       Python's evaluation works, at least one side of an operator needs to be a 
93       SymPy object for the SymPy evaluation to take over. You could write this 
94       as ``x + Rational(1, 2)``, but this is a lot more typing. A shorter 
95       version is ``x + S(1)/2``. Since ``S(1)`` returns ``Integer(1)``, the 
96       division will return a ``Rational`` type, since it will call 
97       ``Integer.__truediv__``, which knows how to return a ``Rational``. 
98    
99       """
100      __slots__ = ()
101  
102      Zero: _Zero
103      One: _One
104      NegativeOne: _NegativeOne
105      Half: _Half
106      Infinity: _Infinity
107      NegativeInfinity: _NegativeInfinity
108      ComplexInfinity: _ComplexInfinity
109      NaN: _NaN
110  
111      # Also allow things like S(5)
112      __call__ = staticmethod(sympify)
113  
114      def __init__(self):
115          self._classes_to_install = {}
116          # Dict of classes that have been registered, but that have not have been
117          # installed as an attribute of this SingletonRegistry.
118          # Installation automatically happens at the first attempt to access the
119          # attribute.
120          # The purpose of this is to allow registration during class
121          # initialization during import, but not trigger object creation until
122          # actual use (which should not happen until after all imports are
123          # finished).
124  
125      def register(self, cls):
126          # Make sure a duplicate class overwrites the old one
127          if hasattr(self, cls.__name__):
128              delattr(self, cls.__name__)
129          self._classes_to_install[cls.__name__] = cls
130  
131      def __getattr__(self, name):
132          """Python calls __getattr__ if no attribute of that name was installed 
133          yet. 
134   
135          Explanation 
136          =========== 
137   
138          This __getattr__ checks whether a class with the requested name was 
139          already registered but not installed; if no, raises an AttributeError. 
140          Otherwise, retrieves the class, calculates its singleton value, installs 
141          it as an attribute of the given name, and unregisters the class."""
142          if name not in self._classes_to_install:
143              raise AttributeError(
144                  "Attribute '%s' was not installed on SymPy registry %s" % (
145                  name, self))
146          class_to_install = self._classes_to_install[name]
147          value_to_install = class_to_install()
148          self.__setattr__(name, value_to_install)
149          del self._classes_to_install[name]
150          return value_to_install
151  
152      def __repr__(self):
153          return "S"
154  
155  S = SingletonRegistry()

158  class Singleton(type):
159      """ 
160      Metaclass for singleton classes. 
161   
162      Explanation 
163      =========== 
164   
165      A singleton class has only one instance which is returned every time the 
166      class is instantiated. Additionally, this instance can be accessed through 
167      the global registry object ``S`` as ``S.<class_name>``. 
168   
169      Examples 
170      ======== 
171   
172          >>> from sympy import S, Basic 
173          >>> from sympy.core.singleton import Singleton 
174          >>> class MySingleton(Basic, metaclass=Singleton): 
175          ...     pass 
176          >>> Basic() is Basic() 
177          False 
178          >>> MySingleton() is MySingleton() 
179          True 
180          >>> S.MySingleton is MySingleton() 
181          True 
182   
183      Notes 
184      ===== 
185   
186      Instance creation is delayed until the first time the value is accessed. 
187      (SymPy versions before 1.0 would create the instance during class 
188      creation time, which would be prone to import cycles.) 
189      """
190      def __init__(cls, *args, **kwargs):
191          cls._instance = obj = Basic.__new__(cls)
192          cls.__new__ = lambda cls: obj
193          cls.__getnewargs__ = lambda obj: ()
194          cls.__getstate__ = lambda obj: None
195          S.register(cls)
196  
197  
198  # Delayed to avoid cyclic import
