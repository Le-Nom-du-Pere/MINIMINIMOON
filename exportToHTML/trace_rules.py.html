82   """ 
83   A note on skip/inline rules: 
84    
85   Dynamo consults this file to determine whether function should be inlined or skipped. 
86    
87   A skip applies at the frame boundary, meaning dynamo either triggers a graph break 
88   at the beginning of the frame or attempts to trace/inline the whole frame. When skipping 
89   a frame, recursively called frames are still traced by dynamo unless also skipped. 
90    
91   Skipfiles (skipped at the file level instead of function level) still apply on a 
92   frame-by-frame boundary as dynamo traces, but apply to all functions in that file. 
93    
94   @skip is a helper decorator that can be applied to your function to cause it to be 
95   included here. 
96    
97   Dynamo skip/inline rules & priorities are defined as follows: 
98   * Inline is the default behavior and will be used unless explicitly skipped. 
99   * Dynamo has two SKIPLIST: BUILTIN_SKIPLIST and THIRDPARTY_SKIPLIST. 
100      * BUILTIN_SKIPLIST contains builtin python modules, such as abc, collections, etc. 
101      * THIRDPARTY_SKIPLIST contains common third party libraries, such as numpy, pandas, etc. 
102  * Functions in these two SKIPLISTs are always skipped, except: 
103      * They have explicitly defined rule in `manual_torch_name_rule_map`; 
104      * The corresponding python module has been put into MOD_INLINELIST. 
105  * PyTorch(torch) is in the BUILTIN_SKIPLIST by default, but there are many cases 
106      where we want inline the functions under torch namespace. 
107      We should specify inline for the functions in `manual_torch_name_rule_map` or 
108      put the corresponding python module into MOD_INLINELIST to make dynamo inline them. 
109  * If you call functions under skipped modules/files, Dynamo will wrap these functions 
110      as SkipFunctionVariable. There are a few functions(e.g, collections.OrderedDict) that 
111      we have special handling at SkipFunctionVariable.call_function. 
112   
113  Overall: *_INLINELIST has precedence over *_SKIPLIST has precedence over DEFAULT (inline) 
114   
115  To figure out what the behavior is, check the following list in order: 
116  * `manual_torch_name_rule_map` (Inline if YES) 
117  * MOD_INLINELIST (Inline if YES) 
118  * BUILTIN_SKIPLIST & THIRDPARTY_SKIPLIST (Skip if YES) 
119  * MOD_SKIPLIST (Skip if YES) 
120  * Inline by default 
121   
122  In general, if you want to force inline a function or module, please consider adding 
123  the function's python module to MOD_INLINELIST first. 
124  Use the `manual_torch_name_rule_map` only when there are other functions under the same module that 
125  you don't want to inline them. 
126  """

3707 This is the main entry point to determine whether an object (function) should be inlined or skipped. 
3708 Let's illustrate the logic with an example: 
3709     @torch.compile 
3710     def f1(x, y): 
3711         ...... 
3712         f2(x, y) 
3713         ...... 
3714  
3715     def f2(x, y): 
3716         ...... 
3717         f3(x, y) 
3718         ...... 
3719  
3720     def f3(x, y): 
3721         ...... 
3722  
3723 There are mainly three call sites of check/check_verbose: 
3724 * The compile region entrance (like function f1), the corresponding code is located at eval_frame.py. 
3725 * When tracing the recursively called functions (like function f2 and f3). 
3726     * Dynamo decides inline/skip every time it encounters a new recursively function call, and the call site 
3727       is in InliningInstructionTranslator.check_inlineable of symbolic_convert.py. 
3728     * If f2 is skipped by Dynamo, when evaluating the frame of f3, Dynamo need the inline/skip check again 
3729       and the call site is in catch_errors_wrapper.catch_errors of convert_frame.py. 
3730 * For global variables and function arguments, Dynamo needs to decide if they are wrapped as SkipFunctionVariable in builder.py. 
3731

3733 or not (f3 is not inlined call if f2 is skipped). Inside of the `check_verbose` function, there are more rules 
3734 to be checked if this `is_inlined_call`.

3736 we don't want to inline the lower level function call (e.g, f3) by default.

3890 def _lookup_inner(
3891     obj,
3892     name=None,
3893     filename=None,
3894     is_direct_call=True,
3895     reasons: Union[None, set[str]] = None,
3896 ):
3897     # Step 1: lookup obj's tracing rule in `torch_name_rule_map`.
3898     # The rules defined in `torch_name_rule_map` mainly includes two parts:
3899     # - Manually defined rules for any functions.
3900     # - The list of torch in graph functions.
3901     try:
3902         can_hash = hashable(obj)
3903     except Exception:
3904         can_hash = False
3905     if not can_hash:
3906         if reasons is not None:
3907             reasons.add("obj is not hashable")
3908         return None
3909     if obj is not None:
3910         if is_aten_op_or_tensor_method(obj):
3911             return TorchInGraphFunctionVariable
3912         rule = get_torch_obj_rule_map().get(obj, None)
3913         if rule is not None:
3914             if reasons is not None:
3915                 reasons.add("get_torch_obj_rule_map")
3916             return rule
3917     elif name is not None and filename is not None and not is_direct_call:
3918         if name.startswith(TORCH_DYNAMO_RESUME_IN_PREFIX):
3919             rule = get_torch_obj_rule_map().get(
3920                 filename + "#" + TORCH_DYNAMO_RESUME_IN_PREFIX, None
3921             )
3922         else:
3923             rule = get_torch_obj_rule_map().get(filename + "#" + name, None)
3924         if rule is not None:
3925             if reasons is not None:
3926                 reasons.add("get_torch_obj_rule_map")
3927             return rule
3928     elif name == "<listcomp>":
3929         if reasons is not None:
3930             reasons.add("inlining frame from list comprehension")
3931         return UserFunctionVariable
3932 
3933     # Step 2: lookup obj's tracing rule by function name.
3934     if is_direct_call:
3935         if name == "patched_init":
3936             if reasons is not None:
3937                 reasons.add("func name is patched_init")
3938             return SkipFunctionVariable
3939         elif name == "__torch_function__" or (
3940             obj and getattr(obj, "__name__", None) == "__torch_function__"
3941         ):
3942             if reasons is not None:
3943                 reasons.add("func name is __torch_function__")
3944             return UserFunctionVariable
3945 
3946     if not is_direct_call:
3947         if name == "__getattr__":
3948             # is_direct_call = False indicates that this is the top-level frame
3949             # being traced (i.e., it is not inlined and not called from
3950             # InliningInstructionTranslator).  Tracing __getattr__ at the top
3951             # level is unlikely because we inline it for
3952             # UserDefinedObjectVariable. This scenario occurs only for
3953             # UnspecializedNNModuleVariable, where Dynamo directly calls
3954             # __getattr__ during trace time, generating LOAD_ATTR bytecode
3955             # without going through the underlying __getattr__ data structures.
3956             # When this optimized bytecode is executed, Dynamo is triggered
3957             # again on the __getattr__ call. Therefore, we skip Dynamo tracing
3958             # in this case.
3959             if reasons is not None:
3960                 reasons.add(
3961                     "Tracing __getattr__ as the top level frame, unsuitable for tracing."
3962                 )
3963             return SkipFunctionVariable
3964 
3965     # Step 3: lookup obj's tracing rule by filename.
3966     if filename is None:
3967         filename = getfile(obj)
3968 
3969     skip_result = check_file(filename, is_direct_call)
3970     if reasons is not None:
3971         reasons.add(skip_result.reason)
3972     if skip_result.skipped:
3973         return SkipFunctionVariable
3974     else:
3975         return UserFunctionVariable
