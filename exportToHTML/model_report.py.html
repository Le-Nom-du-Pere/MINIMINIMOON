23   class ModelReport:
24       r""" 
25       The ModelReport class aims to provide users an easy way to diagnose issues that they run into 
26       with their models. The class works with all traceable GraphModules to help diagnose issues, 
27       though the requirements on the type of model more-so depends on the specific report the user 
28       is trying to generate. With respect to the reports, the ModelReport class is initialized with 
29       a set of Detector classes, each of which generate reports on quantization configuration 
30       issues a use might have. 
31    
32       Currently supports generating reports on: 
33       - Suggestions for per-channel vs. per-tensor quantization (nn.Module) 
34       - Suggestions for dynamic vs static quantization for linear layers (Graph Modules) 
35       - Suggestions for input-weight equalization for linear and conv layers (Graph Modules) 
36       - Suggestions for outlier detection for all layers (Graph Modules) 
37    
38       The ModelReport class has the primary functionality of inserting observers (primarily the ModelReportObserver) 
39       where needed for each detector to gather the information it needs, and then after callibration, the ModelReport 
40       class compiles the report generated by each Detector class into a single report to return to the user. It also 
41       has the capability to remove all the observers it inserted as well. 
42    
43       * :attr:`_model` The model we wish to generate the report for. Must be a traceable GraphModule 
44    
45       * :attr:`_desired_report_detectors` The set of Detectors representing desired reports from the ModelReport class 
46           Make sure that these are all unique types of detectors [do not have more than 1 of the same class] 
47    
48       * :attr:`_desired_detector_names` The set of detector names of the _desired_report_detectors. 
49           This set is generated by calling the get_detector_name() of each detector 
50    
51       * :attr:`_detector_name_to_observer_fqns` The mapping from each detector to fqns of observers of interest 
52           The purpose of this is to keep track of what observers were inserted for each detector, so that they 
53           can be removed at the end if desired 
54    
55       * :attr:`_prepared_flag` A boolean flag that keeps track of whether we have prepared the model or not 
56           This is to ensure we only insert observers once with the ModelReport instance 
57    
58       * :attr:`_removed_observers` A boolean to track if we have removed observers already 
59           The purpose is to ensure we don't attempt to remove observers twice with the same ModelReport 
60           instance. This also allows the functionality where we can generate the report multiple times 
61           as long as we haven't removed the observers yet. 
62    
63       Note: 
64           This class was initially designed to work with the Fx Graph Mode workflow in mind. However, 
65           full functionality is available as long as there is a traceable GraphModule that is being used. 
66           One method to get a traceable GraphModule without going through the Fx workflow is to use 
67           the QuantizationTracer class. 
68    
69       General Flow for Fx workflow: 
70       1.) Initialize ModelReport object with reports of interest by passing in initialized detector objects and model 
71       2.) Prepare your model with prepare_fx 
72       3.) Call model_report.prepare_detailed_calibration to add relevant observers 
73       4.) Callibrate your model with data 
74       5.) Call model_report.generate_report on your model to generate report and optionally remove added observers 
75       Optional 
76           6.) Call model_report.generate_visualizer to get a ModelReportVisualizer instance 
77           7.) To help in parsing report information and debugging, view report info as a: 
78               - Table 
79               - Histogram 
80               - Line plot 
81       8.) Call model_report.generate_qconfigs to generate the qconfigs based on the report suggestions 
82    
83       Example (with QuantizationTracer): 
84           >>> # xdoctest: +SKIP 
85           >>> # get the necessary qconfig 
86           >>> config = PrepareCustomConfig() 
87           >>> skipped_module_names, skipped_module_classes = ( 
88           ...     get_skipped_module_name_and_classes(config, False) 
89           ... ) 
90    
91           >>> # initialize our model and get GraphModule 
92           >>> model = SomeModel() 
93           >>> tracer = QuantizationTracer(skipped_module_names, skipped_module_classes) 
94           >>> graph_module = GraphModule(model, tracer.trace(model)) 
95    
96           >>> # get our set of detectors and ModelReport instance 
97           >>> detector_set = set( 
98           ...     [ 
99           ...         DynamicStaticDetector(tolerance=0.5), 
100          ...         InputWeightEqualizationDetector(ratio_threshold=0.7), 
101          ...     ] 
102          ... ) 
103          >>> tracer_reporter = ModelReport(graph_module, tracer_detector_set) 
104   
105          >>> # now we insert the observers and callibrate the model 
106          >>> tracer_model_with_observers = tracer_reporter.prepare_detailed_calibration() 
107          >>> for i in range(num_callibration_batches): 
108          >>>     example_input = get_callibration_input() 
109          >>>     tracer_model_with_observers(example_input) 
110   
111          >>> # finally we generate the reports and optionally remove the observers we inserted 
112          >>> reports = tracer_reporter.generate_model_report( 
113          ...     remove_inserted_observers=True 
114          ... ) 
115   
116          >>> # Optional: we can generate the qconfig mapping based on the suggestions 
117          >>> qconfigs = model_report.generate_qconfig_mapping() 
118   
119          >>> # Optional: we can generate the equalization mapping based on the suggestions 
120          >>> qconfigs = model_report.generate_equalization_mapping() 
121   
122          >>> # Optional: we get a ModelReportVisualizer instance to do any visualizations desired 
123          >>> model_report_visualizer = tracer_reporter.generate_visualizer() 
124   
125      """

127      def __init__(self, model: GraphModule, desired_report_detectors: set[DetectorBase]):
128          if len(desired_report_detectors) == 0:
129              raise ValueError("Should include at least 1 desired report")
130  
131          # keep track of the model we wish to generate report for
132          self._model: GraphModule = model
133  
134          # keep the reports private so they can't be modified
135          self._desired_report_detectors = desired_report_detectors
136          self._desired_detector_names = {
137              detector.get_detector_name() for detector in desired_report_detectors
138          }
139  
140          # keep a mapping of desired reports to observers of interest
141          # this is to get the readings, and to remove them, can create a large set
142          # this set can then be used to traverse the graph and remove added observers
143          self._detector_name_to_observer_fqns: dict[str, set[str]] = {}
144  
145          # initialize each report to have empty set of observers of interest
146          for desired_report in self._desired_detector_names:
147              self._detector_name_to_observer_fqns[desired_report] = set()
148  
149          # flags to ensure that we can only prepare and remove observers once
150          self._prepared_flag = False
151          self._removed_observers = False
152  
153          # store the reports that we generated for visualization purposes
154          # initially empty since no reports generated
155          self._generated_reports: dict[str, dict] = {}
156

158          """Returns a copy of the desired reports for viewing"""

165      def prepare_detailed_calibration(self) -> GraphModule:
166          r""" 
167          Takes in a graph model and inserts the following observers: 
168          - ModelReportObserver 
169   
170          Each observer is inserted based on the desired_reports into the relevant locations 
171   
172          Right now, each report in self._desired_detector_names has independent insertions 
173              However, if a module already has a Observer of the same type, the insertion will not occur 
174              This is because all of the same type of Observer collect same information, so redundant 
175   
176          Returns the same GraphModule with the observers inserted 
177          """
178  
179          # if already prepared once, cannot prepare again
180          if self._prepared_flag:
181              raise ValueError(
182                  "Already ran preparing detailed callibration. Run the report generation next after callibration."
183              )
184  
185          # loop through each detector, find where placements should be, and keep track
186          insert_observers_fqns: dict[str, Any] = {}
187  
188          for detector in self._desired_report_detectors:
189              # determine observer points for each detector
190              obs_fqn_to_info = detector.determine_observer_insert_points(self._model)
191              # map each insert point to the observer to use
192              insert_observers_fqns.update(obs_fqn_to_info)
193              # update the set of observers this report cares about
194              self._detector_name_to_observer_fqns[detector.get_detector_name()] = set(
195                  obs_fqn_to_info.keys()
196              )
197  
198          # now insert all the observers at their desired locations
199          for observer_fqn in insert_observers_fqns:
200              target_node = insert_observers_fqns[observer_fqn][DETECTOR_TARGET_NODE_KEY]
201              insert_obs = insert_observers_fqns[observer_fqn][DETECTOR_OBS_TO_INSERT_KEY]
202              insert_post = insert_observers_fqns[observer_fqn][DETECTOR_IS_POST_OBS_KEY]
203              observer_args = insert_observers_fqns[observer_fqn][DETECTOR_OBS_ARGS_KEY]
204              self._insert_observer_around_module(
205                  observer_fqn, target_node, insert_obs, observer_args, insert_post
206              )
207  
208          self._prepared_flag = True
209  
210          return self._model

212      def _insert_observer_around_module(
213          self,
214          obs_fqn: str,
215          target_node: torch.fx.node.Node,
216          obs_to_insert: ObserverBase,
217          observer_args: tuple,
218          insert_post: bool,
219      ):
220          r""" 
221          Helper function that inserts the observer into both the graph structure and the module of the model 
222   
223          Args 
224              node_fqn (str): The fully qualified name of the observer we want to insert 
225              target_node (torch.fx.node.Node): The node in model we are inserting observers around 
226              obs_to_insert (ObserverBase): The observer we are inserting around target_node 
227              observer_args (Tuple): The arguments we want to pass into the observer 
228              insert_post (bool): whether this is meant to be a post observer for this node 
229          """
230          # if we are inserting post, then our target node is the next node
231          if insert_post:
232              target_node = target_node.next
233  
234          with self._model.graph.inserting_before(target_node):
235              self._model.add_submodule(obs_fqn, obs_to_insert)
236              self._model.graph.create_node(
237                  op="call_module", target=obs_fqn, args=observer_args
238              )
239  
240          # recompile model after inserts are made
241          self._model.recompile()
242  
243      def _get_node_from_fqn(self, node_fqn: str) -> torch.fx.node.Node:
244          r""" 
245          Takes in a node fqn and returns the node based on the fqn 
246   
247          Args 
248              node_fqn (str): The fully qualified name of the node we want to find in model 
249   
250          Returns the Node object of the given node_fqn otherwise returns None 
251          """
252          node_to_return = None
253          for node in self._model.graph.nodes:
254              # if the target matches the fqn, it's the node we are looking for
255              if node.target == node_fqn:
256                  node_to_return = node
257                  break
258  
259          if node_to_return is None:
260              raise ValueError("The node_fqn is was not found within the module.")
261  
262          # assert for MyPy
263          assert isinstance(node_to_return, torch.fx.node.Node)
264  
265          return node_to_return

267      def generate_model_report(
268          self, remove_inserted_observers: bool
269      ) -> dict[str, tuple[str, dict]]:
270          r""" 
271          Generates all the requested reports. 
272   
273          Note: 
274              You should have callibrated the model with relevant data before calling this 
275   
276          The reports generated are specified by the desired_reports specified in desired_reports 
277   
278          Can optionally remove all the observers inserted by the ModelReport instance 
279   
280          Args: 
281              remove_inserted_observers (bool): True to remove the observers inserted by this ModelReport instance 
282   
283          Returns a mapping of each desired report name to a tuple with: 
284              The textual summary of that report information 
285              A dictionary containing relevant statistics or information for that report 
286   
287          Note: 
288              Throws exception if we try to generate report on model we already removed observers from 
289              Throws exception if we try to generate report without preparing for callibration 
290          """
291          # if we haven't prepped model for callibration, then we shouldn't generate report yet
292          if not self._prepared_flag:
293              raise Exception(  # noqa: TRY002
294                  "Cannot generate report without preparing model for callibration"
295              )
296  
297          # if we already removed the observers, we cannot generate report
298          if self._removed_observers:
299              raise Exception(  # noqa: TRY002
300                  "Cannot generate report on model you already removed observers from"
301              )
302  
303          # keep track of all the reports of interest and their outputs
304          reports_of_interest = {}
305  
306          for detector in self._desired_report_detectors:
307              # generate the individual report for the detector
308              report_output = detector.generate_detector_report(self._model)
309              reports_of_interest[detector.get_detector_name()] = report_output
310  
311          # if user wishes to remove inserted observers, go ahead and remove
312          if remove_inserted_observers:
313              self._removed_observers = True
314              # get the set of all Observers inserted by this instance of ModelReport
315              all_observers_of_interest: set[str] = set()
316              for desired_report in self._detector_name_to_observer_fqns:
317                  observers_of_interest = self._detector_name_to_observer_fqns[
318                      desired_report
319                  ]
320                  all_observers_of_interest.update(observers_of_interest)
321  
322              # go through all_observers_of_interest and remove them from the graph and model
323              for observer_fqn in all_observers_of_interest:
324                  # remove the observer from the model
325                  self._model.delete_submodule(observer_fqn)
326  
327                  # remove the observer from the graph structure
328                  node_obj = self._get_node_from_fqn(observer_fqn)
329  
330                  if node_obj:
331                      self._model.graph.erase_node(node_obj)
332                  else:
333                      raise ValueError("Node no longer exists in GraphModule structure")
334  
335              # remember to recompile the model
336              self._model.recompile()
337  
338          # save the generated reports for visualization purposes
339          saved_reports: dict[str, dict] = {
340              report_name: report_tuple[1]
341              for report_name, report_tuple in reports_of_interest.items()
342          }
343  
344          self._generated_reports = saved_reports
345  
346          # return the reports of interest
347          return reports_of_interest

349      def _is_same_info_for_same_key(self, info_dict_a: dict, info_dict_b: dict) -> bool:
350          r""" 
351          Takes in two dictionaries and ensures that any common keys between the two have the same 
352          values. 
353   
354          Args: 
355              info_dict_a (Dict): First dictionary we wish to compare 
356              info_dict_b (Dict): Second dictionary we wish to compare 
357   
358          Returns True if all shared keys have same values, false otherwise 
359          """
360          # get the set of keys for both
361          dict_a_keys: set = set(info_dict_a.keys())
362          dict_b_keys: set = set(info_dict_b.keys())
363  
364          # get the insersection keys and check if same value for both dicts
365          intersecting_keys: set = dict_a_keys.intersection(dict_b_keys)
366  
367          for key in intersecting_keys:
368              dict_a_val = info_dict_a[key]
369              dict_b_val = info_dict_b[key]
370  
371              # if it's a tensor we have to handle separately
372              if type(dict_a_val) == torch.Tensor:
373                  # if dict_b_val not tensor, automatically false
374                  if (
375                      type(dict_b_val) != torch.Tensor
376                      or sum(dict_a_val != dict_b_val) != 0
377                  ):
378                      return False
379              else:
380                  # for non-tensor vals
381                  if dict_a_val != dict_b_val:
382                      return False
383  
384          # if no non matching shared keys found, return true
385          return True

387      def _reformat_reports_for_visualizer(self) -> OrderedDict:
388          r""" 
389          Takes the generated reports and reformats them into the format that is desired by the 
390          ModelReportVisualizer 
391   
392          Returns an OrderedDict mapping module_fqns to their features 
393          """
394          # we want to reorder and reformat the information so it is ordered in terms of order
395          # found in the model
396  
397          # first create new dict with all modules as keys and features under respective module
398          module_fqns_to_features: dict[str, dict] = {}
399  
400          for report_name in self._generated_reports:
401              # get mod -> feature dict and go through
402              module_info = self._generated_reports[report_name]
403  
404              for module_fqn in module_info:
405                  # check if already in our accumulation dict
406                  if module_fqn in module_fqns_to_features:
407                      # we merge all the features together
408                      new_info: dict = module_info[module_fqn]
409                      present_info: dict = module_fqns_to_features[module_fqn]
410  
411                      # merge them together into the new unioned dict
412                      # same features keys -> same info, so okay if override
413  
414                      # do safety check to make sure shared keys have same info
415                      if self._is_same_info_for_same_key(new_info, present_info):
416                          module_fqns_to_features[module_fqn] = {
417                              **new_info,
418                              **present_info,
419                          }
420                      else:
421                          error_str = "You have the same key with different values across detectors. "
422                          error_str += "Someone incorrectly implemented a detector with conflicting keys to existing detectors."
423                          raise ValueError(error_str)
424                  else:
425                      # we just set it
426                      module_fqns_to_features[module_fqn] = module_info[module_fqn]
427  
428          # our ordered dict so that modules can be ordered in order of how they appear in model
429          features_by_module: OrderedDict[str, dict] = OrderedDict()
430  
431          # we loop through modules in graph in order
432          for fqn, _module in self._model.named_modules():
433              # find that fqn in fqns_to_features
434              if fqn in module_fqns_to_features:
435                  # add it to our ordered dict
436                  features_by_module[fqn] = module_fqns_to_features[fqn]
437  
438          # return the ordered dict of info we created
439          return features_by_module

441      def generate_visualizer(self) -> ModelReportVisualizer:
442          r""" 
443          Generates a ModelReportVisualizer instance using the reports generated 
444          by the generate_model_report() method. 
445   
446          Returns the generated ModelReportVisualizer instance initialized 
447   
448          Note: 
449              Throws exception if attempt to get visualizers without generating report 
450          """
451          # check if user has generated reports at least once
452          if len(self._generated_reports) == 0:
453              raise Exception(  # noqa: TRY002
454                  "Unable to generate visualizers without first generating reports"
455              )
456  
457          # get the ordered dict mapping modules to their full set of collected features / stats
458          module_fqns_to_features: OrderedDict = self._reformat_reports_for_visualizer()
459  
460          # create and return ModelReportVisualizer instance
461          visualizer: ModelReportVisualizer = ModelReportVisualizer(
462              module_fqns_to_features
463          )
464  
465          return visualizer

467      def _generate_qconfig_mapping_helper(
468          self,
469          detector_qconfig_info_combined: dict[str, DetectorQConfigInfo],
470          generation_function: Callable,
471      ) -> QConfigMapping:
472          r""" 
473          This helper takes in the compiled detector qconfig info that 
474          has been compiled together and merges it into a QConfigMapping 
475          """
476          # keep track of the qconfigmapping
477          qconfig_mapping = QConfigMapping()
478  
479          # loop through each module / fqn and attempt to create QConfigMapping
480          for fqn, module in self._model.named_modules():
481              # if we have a qconfig info for this module
482              if fqn in detector_qconfig_info_combined:
483                  qconfig_info_compiled = detector_qconfig_info_combined[fqn]
484  
485                  # now generate the qconfig and add it to the mapping
486                  generated_qconfig = generation_function(qconfig_info_compiled, module)
487  
488                  # add to our config
489                  qconfig_mapping.set_module_name(fqn, generated_qconfig)
490  
491          # return compiled mapping
492          return qconfig_mapping

529      def _generate_module_fqn_to_detector_info_mapping(
530          self, update_qconfig_info_function: Callable
531      ) -> dict[str, DetectorQConfigInfo]:
532          r""" 
533          Generates a QConfigMapping based on the suggestions of the 
534          ModelReport API. The generated mapping encompasses all the 
535          different types of feedback from the different detectors 
536          all into one place. 
537   
538          These configs are based on the suggestions provided by the ModelReport API 
539          and can only be generated once the reports have been generated. 
540   
541          Args: 
542              update_qconfig_info_function (Callable) takes in a function that takes in two DetectorQConfigInfo 
543              and updates the one that is being compiled 
544   
545          Returns a Dict mapping module_fqns to DetectorQConfigInfo objects 
546   
547          Note: 
548              Throws exception if we try to generate mapping on model we already removed observers from 
549              Throws exception if we try to generate mapping without preparing for callibration 
550          """
551          # if we haven't prepped model for callibration, then we shouldn't generate mapping yet
552          if not self._prepared_flag:
553              raise Exception(  # noqa: TRY002
554                  "Cannot generate report without preparing model for callibration"
555              )
556  
557          # if we already removed the observers, we cannot mapping
558          if self._removed_observers:
559              raise Exception(  # noqa: TRY002
560                  "Cannot generate report on model you already removed observers from"
561              )
562  
563          # keep track of qconfig info for each module across detectors
564          detector_qconfig_info_combined: dict[str, DetectorQConfigInfo] = {}
565  
566          for detector in self._desired_report_detectors:
567              # get the info from the detector
568              detector_info: dict[str, DetectorQConfigInfo] = detector.get_qconfig_info(
569                  self._model
570              )
571  
572              # we go through the modules
573              for module_fqn in detector_info:
574                  # see if we already have info on it
575                  if module_fqn in detector_qconfig_info_combined:
576                      # we combine the current options with what is there
577                      current_options = detector_qconfig_info_combined[module_fqn]
578                      detector_options = detector_info[module_fqn]
579  
580                      update_qconfig_info_function(current_options, detector_options)
581                  else:
582                      # we just use this for now
583                      detector_qconfig_info_combined[module_fqn] = detector_info[
584                          module_fqn
585                      ]
586  
587          return detector_qconfig_info_combined

589      def generate_qconfig_mapping(self) -> QConfigMapping:
590          r""" 
591          Generates a QConfigMapping based on the suggestions of the 
592          ModelReport API. The generated mapping encompasses all the 
593          different types of feedback from the different detectors 
594          all into one place. 
595   
596          These configs are based on the suggestions provided by the ModelReport API 
597          and can only be generated once the reports have been generated. 
598   
599          Returns a QConfigMapping for the quantization configuration 
600   
601          Note: 
602              Throws exception if we try to generate mapping on model we already removed observers from 
603              Throws exception if we try to generate mapping without preparing for callibration 
604          """
605          # get the mapping info
606          detector_qconfig_info_combined = (
607              self._generate_module_fqn_to_detector_info_mapping(
608                  self._update_detector_quantizaiton_qconfig_info
609              )
610          )
